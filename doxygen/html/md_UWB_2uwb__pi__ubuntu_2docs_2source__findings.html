Source findings from uploaded Qorvo files

These are the implementation-relevant findings used by the starter code and to-do list.

Range is available

SDK/Documentation/uwb-stack/uwb-uci-messages-api-R12.7.0-405.pdf, section 5.7, states that SESSION_INFO_NTF sends ranging results to the host. The TWR measurement table includes Distance as a 2-octet field.

The SDK Python parser confirms this in:

SDK/Tools/uwb-qorvo-tools/lib/uwb-uci/uci/qorvo_msg.py

RangingTwrData.__init__() parses:

self.distance = float(b.pop_uint(2)) # distance in cm

Range rate is not directly available

The same TWR measurement parser and UCI message table include distance, AoA, RSSI, NLoS, status, and slot index, but no range-rate/radial-velocity field. The starter gateway therefore computes range rate on the Pi from successive valid distance samples and marks it as derived.

Default FiRa settings

SDK/Documentation/DeveloperManual/DWM3001CDK_Developer_Manual_QM33SDK-1.1.1.pdf, section 7.6.2, shows the default commands:

INITF -CHAN=9 -PRFSET=BPRF4 -SLOT=2400 -BLOCK=200 -ROUND=25 -RRU=DSTWR -ID=42 -VUPPER=01:02:03:04:05:06:07:08 -ADDR=0 -PADDR=1
RESPF -CHAN=9 -PRFSET=BPRF4 -SLOT=2400 -BLOCK=200 -ROUND=25 -RRU=DSTWR -ID=42 -VUPPER=01:02:03:04:05:06:07:08 -ADDR=1 -PADDR=0

The starter gateway uses these same defaults through the UCI app config API.

UCI firmware is the correct integration point

The Developer Manual says the UCI project provides a way to fully control the FiRa UWB stack over the UCI interface, and the UCI task transfers UCI messages from UART/USB to the UWB stack at runtime.

Low-level DW3xxx driver porting is not the first choice here

DW3XXX_Software_API_Guide_4p12.pdf explains that the low-level driver expects platform functions like writetospi(), readfromspi(), decamutexon(), decamutexoff(), and an interrupt path that calls dwt_isr(). It also warns that the driver APIs are not re-entrant except for the interrupt-handling provisions. Since the Pi is available and the UWB board is connected over USB/UART, the UCI stack is the cleaner integration path.
