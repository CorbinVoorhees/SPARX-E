This repository is the Software Development Kit (SDK) for the \href{https://www.unitree.com/LiDAR}{\texttt{ Unitree L2}} Li\+DAR.

You can use the code interfaces in this repository to obtain point cloud and IMU data from our Li\+DAR, as well as set and get relevant configuration parameters of the lidar.

We provide several common interfaces for the Li\+DAR\+:
\begin{DoxyItemize}
\item The original C++ based SDK\+: \doxysectlink{md_src_2unilidar__sdk2_2unitree__lidar__sdk_2README}{unitree\+\_\+lidar\+\_\+sdk}{0}
\item The package for parsing and publishing Li\+DAR data in the ROS environment\+: unitree\+\_\+lidar\+\_\+ros
\item The package for parsing and publishing Li\+DAR data in the ROS2 environment\+: \doxysectlink{md_src_2unilidar__sdk2_2unitree__lidar__ros2_2src_2unitree__lidar__ros2_2README}{unitree\+\_\+lidar\+\_\+ros2}{0}
\end{DoxyItemize}\hypertarget{md_src_2unilidar__sdk2_2README_autotoc_md6}{}\doxysection{\texorpdfstring{2. Coordinate System Definition}{2. Coordinate System Definition}}\label{md_src_2unilidar__sdk2_2README_autotoc_md6}
  

The coordinate system of this Li\+DAR is defined as shown in the above figure, which meets the definition of a right-\/handed coordinate system. Let the Li\+DAR point cloud coordinate system be L, and the IMU coordinate system be I.

The origin of the Li\+DAR point cloud coordinate system is located at the center of the bottom mounting surface of the Li\+DAR. Its +X-\/axis is opposite to the direction of the bottom cable outlet, its +Y-\/axis is obtained by rotating the +X-\/axis counterclockwise by 90 degrees, and its +Z-\/axis is perpendicular to the bottom surface.

The three coordinate axes of the IMU coordinate system are parallel to the corresponding coordinate axes of the point cloud coordinate system, and the two only have a translation of the origin position. The origin of the IMU coordinate system in the Li\+DAR point cloud coordinate system is (in meters)\+: \mbox{[}-\/0.\+007698, -\/0.\+014655, 0.\+00667\mbox{]}.

According to the standard transformation matrix method, the pose transformation from the Li\+DAR point cloud coordinate system L to the IMU coordinate system I is\+:

\$\$ T\+\_\+\{LI\} = \textbackslash{}begin\{bmatrix\} 1 \& 0 \& 0 \& -\/0.\+007698 \textbackslash{} 0 \& 1 \& 0 \& -\/0.\+014655 \textbackslash{} 0 \& 0 \& 1 \& 0.\+00667 \textbackslash{} 0 \& 0 \& 0 \& 1 \textbackslash{} \textbackslash{}end\{bmatrix\} \$\$\hypertarget{md_src_2unilidar__sdk2_2README_autotoc_md7}{}\doxysection{\texorpdfstring{3. \hypertarget{md_src_2unilidar__sdk2_2README_autotoc_md9}{}\doxysubsection{\texorpdfstring{3.\+2 Configuring Work Mode}{3.2 Configuring Work Mode}}\label{md_src_2unilidar__sdk2_2README_autotoc_md9}
The Li\+DAR can be configured in various working modes by default, including standard FOV or wide-\/angle FOV, 3D mode or 2D mode, IMU enabled or disabled, Ethernet or serial connection, etc.

We can configure the working mode through the host computer, or we can also configure the working mode through the following interface in {\ttfamily \doxylink{unitree__lidar__sdk_8h}{unitree\+\_\+lidar\+\_\+sdk.\+h}}\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{virtual\ void\ setLidarWorkMode(uint32\_t\ mode)\ =\ 0;}

\end{DoxyCode}


Setting the working mode is implemented through a {\ttfamily uint32\+\_\+t} integer variable, with each bit corresponding to the switching of a function. According to the bit position from low to high, the corresponding functions for positions 0 or 1 are shown in the following table\+:

\tabulinesep=1mm
\begin{longtabu}spread 0pt [c]{*{4}{|X[-1]}|}
\hline
\PBS\centering \cellcolor{\tableheadbgcolor}\textbf{ Bit Position   }&\PBS\centering \cellcolor{\tableheadbgcolor}\textbf{ Function   }&\PBS\centering \cellcolor{\tableheadbgcolor}\textbf{ Value 0   }&\PBS\centering \cellcolor{\tableheadbgcolor}\textbf{ Value 1    }\\\cline{1-4}
\endfirsthead
\hline
\endfoot
\hline
\PBS\centering \cellcolor{\tableheadbgcolor}\textbf{ Bit Position   }&\PBS\centering \cellcolor{\tableheadbgcolor}\textbf{ Function   }&\PBS\centering \cellcolor{\tableheadbgcolor}\textbf{ Value 0   }&\PBS\centering \cellcolor{\tableheadbgcolor}\textbf{ Value 1    }\\\cline{1-4}
\endhead
0   &Switch between standard FOV and wide-\/angle FOV   &Standard FOV (180°)   &Wide-\/angle FOV (192°)    \\\cline{1-4}
1   &Switch between 3D and 2D measurement modes   &3D measurement mode   &2D measurement mode    \\\cline{1-4}
2   &Enable or disable IMU   &Enable IMU   &Disable IMU    \\\cline{1-4}
3   &Switch between Ethernet mode and serial mode   &Ethernet mode   &Serial mode    \\\cline{1-4}
4   &Switch between lidar power-\/on default start mode   &Power on and start automatically   &Power on and wait for start command without rotation    \\\cline{1-4}
5-\/31   &Reserved   &Reserved   &Reserved   \\\cline{1-4}
\end{longtabu}


The common usage mode is standard FOV + 3D measurement + enable IMU + power on self-\/start, which means these bit positions are all kept at 0. We only need to determine whether to use Ethernet connection mode or serial connection mode. For example, in the following sample program,
\begin{DoxyItemize}
\item If using Ethernet connection mode, configure the work mode to 0 (i.\+e., all bit positions of the integer variable are equal to 0)
\item If using serial connection mode, configure the work mode to 8 (i.\+e., the third bit position of the integer variable is equal to 1, and the other 0-\/31 bit positions are all equal to 0)
\end{DoxyItemize}

The default factory lidar work mode is 0, i.\+e., Ethernet communication mode.\hypertarget{md_src_2unilidar__sdk2_2README_autotoc_md10}{}\doxysubsection{\texorpdfstring{3.\+3 Running with Ethernet}{3.3 Running with Ethernet}}\label{md_src_2unilidar__sdk2_2README_autotoc_md10}

If you need to modify the default IP address of the Li\+DAR, you can refer to the user manual to use our host computer to make the changes.\hypertarget{md_src_2unilidar__sdk2_2README_autotoc_md11}{}\doxysubsection{\texorpdfstring{3.\+4 Running with Serial Port}{3.4 Running with Serial Port}}\label{md_src_2unilidar__sdk2_2README_autotoc_md11}
Note that the Li\+DAR is shipped with Ethernet communication mode by default. If you have not yet switched the communication mode to serial communication mode, you will need to use an Ethernet connection to communicate with the Li\+DAR first, so that you can then switch its communication method to serial communication mode. You can use our host computer to make the switch, or you can also refer to section \mbox{[}4.\+2\mbox{]}, use the default Ethernet communication example program {\ttfamily \doxylink{example__lidar__udp_8cpp}{example\+\_\+lidar\+\_\+udp.\+cpp}} to communicate with the lidar first, and set the lidar work mode to serial communication mode ({\ttfamily work\+Mode=8}), then power off and restart the Li\+DAR.

After confirming that the current Li\+DAR is in serial communication mode, we connect the Li\+DAR to the computer using a serial cable, and then confirm the serial port name of the Li\+DAR, which defaults to the following value\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{"{}/dev/ttyACM0"{}}

\end{DoxyCode}


You can also modify the default serial port name and correspondingly modify it in the example code {\ttfamily \doxylink{example__lidar__serial_8cpp}{example\+\_\+lidar\+\_\+serial.\+cpp}}.
