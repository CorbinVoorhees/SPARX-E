The Developer Manual says the UCI project provides a way to fully control the Fi\+Ra UWB stack over the UCI interface, and the UCI task transfers UCI messages from UART/\+USB to the UWB stack at runtime.\hypertarget{md_UWB_2uwb__pi__ubuntu_2docs_2source__findings_autotoc_md59}{}\doxysection{\texorpdfstring{Low-\/level DW3xxx driver porting is not the first choice here}{Low-level DW3xxx driver porting is not the first choice here}}\label{md_UWB_2uwb__pi__ubuntu_2docs_2source__findings_autotoc_md59}
{\ttfamily DW3\+XXX\+\_\+\+Software\+\_\+\+API\+\_\+\+Guide\+\_\+4p12.\+pdf} explains that the low-\/level driver expects platform functions like {\ttfamily writetospi()}, {\ttfamily readfromspi()}, {\ttfamily decamutexon()}, {\ttfamily decamutexoff()}, and an interrupt path that calls {\ttfamily dwt\+\_\+isr()}. It also warns that the driver APIs are not re-\/entrant except for the interrupt-\/handling provisions. Since the Pi is available and the UWB board is connected over USB/\+UART, the UCI stack is the cleaner integration path.
