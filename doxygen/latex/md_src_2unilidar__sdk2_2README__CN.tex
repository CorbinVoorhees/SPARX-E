这个仓库是用于\href{https://www.unitree.com/LiDAR}{\texttt{ Unitree L2}} 激光雷达的软件开发包（\+SDK）。

您可以使用这个仓库中的代码接口来获取我们的激光雷达测量的点云\+IMU等数据，以及设置和获取激光雷达的相关配置参数。

我们为激光雷达提供了几种常用的接口：
\begin{DoxyItemize}
\item 原始的基于\+C++的\+SDK：\doxysectlink{md_src_2unilidar__sdk2_2unitree__lidar__sdk_2README}{unitree\+\_\+lidar\+\_\+sdk}{0}
\item 用于在\+ROS环境下解析和发布激光雷达数据的软件包：unitree\+\_\+lidar\+\_\+ros
\item 用于在\+ROS2环境下解析和发布激光雷达数据的软件包：\doxysectlink{md_src_2unilidar__sdk2_2unitree__lidar__ros2_2src_2unitree__lidar__ros2_2README}{unitree\+\_\+lidar\+\_\+ros2}{0}
\end{DoxyItemize}\hypertarget{md_src_2unilidar__sdk2_2README__CN_autotoc_md25}{}\doxysection{\texorpdfstring{2. 坐标系定义}{2. 坐标系定义}}\label{md_src_2unilidar__sdk2_2README__CN_autotoc_md25}
  

本激光雷达的坐标系定义如上图所示，它满足右手坐标系的定义。设激光雷达点云坐标系为\+L，记\+IMU坐标系为\+I。

激光雷达点云坐标系的原点位于激光雷达底部安装面的中心，其+\+X轴与底部线缆出线方向相反，其+\+Y轴是通过将+\+X轴逆时针旋转90度得到的，其+\+Z轴垂直底面向上。

IMU坐标系的三个坐标轴方向与点云坐标系的对应坐标轴保持平行，二者仅存在原点位置的平移。\+IMU坐标系原点位于激光雷达点云坐标系的位置为（单位为米）：\mbox{[}-\/0.\+007698, -\/0.\+014655, 0.\+00667\mbox{]}。

按照标准的变换矩阵方式，从激光雷达点云坐标系\+L到\+IMU坐标系为\+I的位姿变换为：

\$\$ T\+\_\+\{LI\} = \textbackslash{}begin\{bmatrix\} 1 \& 0 \& 0 \& -\/0.\+007698 \textbackslash{} 0 \& 1 \& 0 \& -\/0.\+014655 \textbackslash{} 0 \& 0 \& 1 \& 0.\+00667 \textbackslash{} 0 \& 0 \& 0 \& 1 \textbackslash{} \textbackslash{}end\{bmatrix\} \$\$\hypertarget{md_src_2unilidar__sdk2_2README__CN_autotoc_md26}{}\doxysection{\texorpdfstring{3. 如何使用\+C++ SDK}{3. 如何使用C++ SDK}}\label{md_src_2unilidar__sdk2_2README__CN_autotoc_md26}

\hypertarget{md_src_2unilidar__sdk2_2README__CN_autotoc_md28}{}\doxysubsection{\texorpdfstring{3.\+2 配置工作模式}{3.2 配置工作模式}}\label{md_src_2unilidar__sdk2_2README__CN_autotoc_md28}
激光雷达默认可以被配置为多种工作模式，包括标准\+FOV或广角\+FOV、3\+D模式或2\+D模式、\+IMU使能或者关闭、以太网或串口连接等。

我们可以可以通过上位机配置工作模式，或者也可使通过{\ttfamily \doxylink{unitree__lidar__sdk_8h}{unitree\+\_\+lidar\+\_\+sdk.\+h}}中的如下接口来配置工作模式： 
\begin{DoxyCode}{0}
\DoxyCodeLine{virtual\ void\ setLidarWorkMode(uint32\_t\ mode)\ =\ 0;}

\end{DoxyCode}


设置工作模式通过一个{\ttfamily uint32\+\_\+t}的整型变量实现，其每一个bit位对应一个功能的切换。按照\+Bit位置从低到高，对应位置为0或1对应的功能如下表所示：

\endfirsthead
\hline
\endfoot
\hline
\PBS\centering \cellcolor{\tableheadbgcolor}\textbf{ 比特位   }&\PBS\centering \cellcolor{\tableheadbgcolor}\textbf{ 功能   }&\PBS\centering \cellcolor{\tableheadbgcolor}\textbf{ 取值0   }&\PBS\centering \cellcolor{\tableheadbgcolor}\textbf{ 取值1    }\\\cline{1-4}
\endhead
0   &切换标准\+FOV和广角\+FOV   &标准\+FOV(180°)   &广角\+FOV(192°)    \\\cline{1-4}

5-\/31   &保留   &保留   &保留   \\\cline{1-4}
\end{longtabu}


常用的使用模式为标准\+FOV+3\+D测量+使能\+IMU+上电自启动，即这几个bit位置均保持为0即可，我们只需要确定是使用网口连接模式还是串口连接模式。例如，在下面的示例程序中，
\begin{DoxyItemize}
\item 如果使用网口连接模式，则将工作模式配置为0（即整型变量的所有bit位置均等于0）
\item 如果使用串口连接模式，则将工作模式配置为8（即整型变量的第3个bit位置等于1，其余0-\/31位的bit位置均等于0）
\end{DoxyItemize}

默认出厂的激光雷达工作模式为0，即网口以太网的通信模式。\hypertarget{md_src_2unilidar__sdk2_2README__CN_autotoc_md29}{}\doxysubsection{\texorpdfstring{3.\+3 使用网口运行}{3.3 使用网口运行}}\label{md_src_2unilidar__sdk2_2README__CN_autotoc_md29}

在确认当前激光雷达为串口通信模式后，我们使用串口线将激光雷达连接到计算机，然后确认激光雷达的串口名称，默认为如下值：
